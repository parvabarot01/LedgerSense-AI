# LedgerSense

LedgerSense is an AI-powered financial anomaly-detection and reconciliation copilot for finance and accounting teams. It ingests transaction and ledger data (bank feeds, payment processor payouts, internal ledgers), automatically detects duplicates, statistical outliers, reconciliation mismatches, and timing gaps, and explains every flagged item in plain language, grounded in the actual data, with a suggested resolution. Every resolution runs through a maker-checker approval workflow with a full audit trail, so nothing is marked resolved without a second signature.

Built to run entirely on free-tier infrastructure: Next.js on Vercel, Supabase for data and auth, Upstash for async job processing, and Groq for the LLM explanation layer.

## Key Features

- **Multi-tenant ingestion & ledger model**: upload transaction data from bank feeds, processor payouts, and internal ledgers as CSV, with a column mapping step that normalizes varying export formats into a single internal ledger model. Data is organized into reconciliation sets and isolated per tenant using Postgres Row-Level Security.
- **Detection engine with four rule types**:
  - Duplicate detection (matching amount, counterparty, and date window)
  - Statistical outlier detection (deviation from historical distribution per counterparty/category)
  - Reconciliation mismatch detection (one-sided entries or value mismatches between sources)
  - Timing gap detection (matched transactions that book outside the expected settlement window)
- **AI-grounded explanations**: every detected exception gets an explanation generated by an LLM (Groq, Llama 3.3 70B), grounded in the actual flagged rows and the specific rule that triggered the flag, with a suggested resolution rather than a generic summary.
- **Maker-checker governance**: a maker proposes a resolution, a distinct checker must approve or reject it before anything is considered resolved. Every state transition is written to an immutable audit log, exportable for compliance review.
- **Executive dashboard**: a rolled-up view of reconciliation health (match rate, open exceptions by severity, time-to-resolution trend) for anyone who needs the status without reading raw ledger rows.

## Tech Stack

- **Framework:** Next.js 14 (App Router), TypeScript, Tailwind CSS
- **Hosting:** Vercel
- **Database / Auth / Storage:** Supabase (Postgres, Auth, Storage, Row-Level Security)
- **Async jobs:** Upstash Redis + QStash
- **Rate limiting:** Upstash Ratelimit
- **Detection math:** simple-statistics
- **AI explanation layer:** Groq (Llama 3.3 70B)
- **CSV parsing:** PapaParse
- **Validation:** Zod
- **Charts:** Recharts
- **Testing:** Vitest, Testing Library

## Local Setup

### Prerequisites

- Node.js 18+
- A Supabase project (free tier)
- An Upstash Redis + QStash account (free tier)
- A Groq API key

### Steps

1. Clone the repository:
   ```bash
   git clone <repo-url>
   cd ledgersense
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Copy the environment template and fill in your credentials:
   ```bash
   cp .env.example .env.local
   ```
   Fill in:
   - `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` (Supabase project settings)
   - `SUPABASE_SERVICE_ROLE_KEY` (Supabase project settings, server-side only)
   - `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` (Upstash Redis dashboard)
   - `QSTASH_TOKEN`, `QSTASH_CURRENT_SIGNING_KEY`, `QSTASH_NEXT_SIGNING_KEY` (Upstash QStash dashboard)
   - `GROQ_API_KEY` (Groq console)

4. Run the database migrations against your Supabase project:
   ```bash
   npx supabase link --project-ref <your-project-ref>
   npx supabase db push
   ```
   Migration files live in `supabase/migrations/`.

5. Start the dev server:
   ```bash
   npm run dev
   ```
   The app runs at [http://localhost:3000](http://localhost:3000).

## Notes

LedgerSense is built to operate entirely within free-tier limits (Vercel, Supabase, Upstash, Groq). Rate limiting is applied on detection and LLM-triggering routes to stay within those limits as usage grows.
